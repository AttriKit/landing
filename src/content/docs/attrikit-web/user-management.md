用户管理是 AttriKit Web 的管理员功能，用于管理团队成员的账号和访问权限。通过用户管理，管理员可以添加新用户、分配角色、重置密码等操作，确保团队成员能够安全、高效地使用系统。

用户管理页面提供完整的用户生命周期管理功能，帮助管理员高效管理团队成员的访问权限。

**在这之前**

> 用户管理是管理员专属功能。只有 Admin 角色的用户才能访问此页面。如果您没有看到「用户管理」选项，请联系系统管理员。

## 权限系统（RBAC）

AttriKit 采用基于角色的访问控制（Role-Based Access Control，RBAC）系统，通过分配不同的角色来控制用户权限。

**角色说明**

| 角色 | 权限范围 | 适用人员 |
|------|----------|----------|
| **Admin（管理员）** | 拥有系统最高权限，可以管理用户、应用和所有监测链接 | 系统管理员、技术负责人 |
| **User（普通用户）** | 可以管理拥有权限的应用下的所有监测链接，不可管理用户和应用 | 运营人员、投放人员 |

**权限对比表**

| 功能模块 | Admin | User |
|----------|-------|------|
| 用户管理 | ✅ | ❌ |
| 应用管理 | ✅ | ❌ |
| 个人中心 | ✅ | ✅ |
| 查看监测链接 | ✅ | 指定应用 |
| 创建监测链接 | ✅ | 指定应用 |
| 编辑监测链接 | ✅ | 指定应用 |
| 删除监测链接 | ✅ | 指定应用 |

## 操作指南

### 步骤一：进入用户管理页面

1. 登录 AttriKit Web 后台
2. 点击页面右上角的个人头像
3. 在下拉菜单中选择「用户管理」
4. 进入用户管理页面

> 如果未看到「用户管理」选项，可能是当前登录用户没有用户管理权限，请联系管理员处理。

![用户管理入口](/images/docs/attrikit-web/profile-menu.webp)

### 步骤二：查看用户列表

用户管理页面展示所有已添加的用户：

- **用户名** - 用户的名称
- **账号** - 用户的登录账号
- **角色** - 用户拥有的角色（Admin/User）
- **操作按钮** - 编辑、重置密码、删除用户的快捷入口

![用户管理页面](/images/docs/attrikit-web/user-management.webp)

### 步骤三：添加新用户

1. 点击页面右上角的「添加用户」按钮
2. 填写用户基本信息
3. 选择用户角色
4. 设置初始密码（可选择随机生成或自定义）
5. 点击「保存」完成创建

![新建用户](/images/docs/attrikit-web/add-user.webp)

### 步骤四：编辑用户信息

1. 在用户列表中找到目标用户
2. 点击「操作」列中的「编辑」按钮
3. 修改需要更新的信息（如名称、角色等）
4. 点击「保存」提交更改

### 步骤五：重置用户密码

1. 在用户列表中找到目标用户
2. 点击「操作」列中的「重置密码」按钮
3. 点击「保存」完成重置

> 用户也可以通过「个人中心」自行重置密码，无需管理员操作。

### 步骤六：删除用户

1. 在用户列表中找到目标用户
2. 点击「操作」列中的「删除」按钮
3. 确认删除操作

> 删除用户后，该用户将无法登录系统，但不会影响其创建的历史数据。

## 密码安全要求

为了确保账号安全，AttriKit 对用户密码有明确的安全要求：

| 要求 | 说明 | 示例 |
|------|------|------|
| **长度** | 12~32 位字符 | 至少 12 个字符 |
| **大写字母** | 必须包含至少一个大写字母 | A-Z |
| **小写字母** | 必须包含至少一个小写字母 | a-z |
| **数字** | 必须包含至少一个数字 | 0-9 |

**密码示例**

✅ **符合要求的密码：**
- `Abc123456789` - 包含大小写字母和数字，长度 12
- `MySecurePass123` - 包含大小写字母和数字，长度 15

❌ **不符合要求的密码：**
- `password` - 缺少大写字母和数字
- `12345678` - 缺少大小写字母
- `Abc123` - 长度不足 12 位

**密码管理建议**

- **首次登录修改** - 建议用户首次登录后立即修改初始密码
- **定期更换** - 建议每 3-6 个月更换一次密码
- **避免复用** - 不要使用与其他系统相同的密码
- **安全存储** - 请勿将密码通过不安全的方式（如邮件、聊天工具）发送

## 最佳实践

**实践 1：角色分配原则**

- **最小权限原则** - 只分配用户完成工作所需的最小权限
- **定期审查** - 定期检查用户角色，及时调整权限
- **职责分离** - 关键操作建议由不同用户分别管理

**实践 2：用户管理建议**

- **及时清理** - 离职用户的账号应及时删除或禁用
- **实名制** - 建议使用真实姓名作为用户名称，便于识别
- **密码策略** - 定期提醒用户更换密码

**实践 3：安全注意事项**

- **初始密码** - 创建用户后，请务必通过安全渠道告知密码
- **密码不可见** - 用户创建完成后，密码将不可再次查看
- **重置密码** - 用户遗忘密码时，可通过「重置密码」功能处理

## 常见问题

### Q1：为什么我看不到用户管理选项？

**A：** 用户管理是管理员专属功能。只有 Admin 角色的用户才能访问，请联系系统管理员为您分配相应权限。

### Q2：可以修改用户名吗？

**A：** 可以。通过编辑用户功能可以修改用户名。

### Q3：用户被删除后，其创建的监测链接会怎样？

**A：** 用户被删除后，其创建的监测链接和历史数据仍会保留，不会受到影响。

### Q4：忘记管理员密码怎么办？

**A：** 如果您忘记了管理员密码，请联系技术支持团队进行密码重置。

### Q5：可以创建多个管理员账号吗？

**A：** 可以。您可以根据需要创建多个 Admin 角色的用户账号。

### Q6：用户如何自行修改密码？

**A：** 用户可以登录后进入「个人中心」，在密码修改区域自行更新密码，无需管理员操作。
