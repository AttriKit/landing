## 转化事件配置

转化事件是归因分析的核心，正确配置才能准确衡量广告效果。

## 什么是转化事件

转化事件是您希望追踪的用户行为，例如：
- 应用下载
- 注册完成
- 首次购买
- 付费订阅
- 关键页面访问

## 创建转化事件

### 基本信息配置

1. 进入 **事件管理** 页面
2. 点击 **创建事件**
3. 填写基本信息：
   - **事件名称**：唯一标识符（如 `app_install`）
   - **显示名称**：后台显示的名称（如 "应用下载"）
   - **事件描述**：详细说明事件含义
   - **事件类型**：选择预定义类型或自定义

### 事件类型预定义

| 事件类型 | 触发时机 | 典型场景 |
|----------|----------|----------|
| app_install | 应用首次启动 | 应用下载广告 |
| register | 用户完成注册 | 注册优化广告 |
| purchase | 完成支付 | 电商转化广告 |
| add_to_cart | 加入购物车 | 购物意向分析 |
| page_view | 页面访问 | 品牌曝光分析 |
| custom | 自定义行为 | 特定业务需求 |

## 事件参数配置

### 标准参数

系统会自动捕获以下参数：

- **event_time**：事件发生时间
- **device_id**：设备标识
- **user_id**：用户标识
- **ip_address**：IP 地址
- **user_agent**：浏览器信息

### 自定义参数

您可以根据业务需求添加自定义参数：

```javascript
// 示例：添加订单金额参数
attrikit.track('purchase', {
  order_id: 'ORD123456',
  amount: 299.00,
  currency: 'CNY',
  product_category: 'electronics'
});
```

### 参数类型

支持以下参数类型：
- **字符串**（string）：文本内容
- **数值**（number）：数字值，可用于计算
- **布尔值**（boolean）：是/否值
- **数组**（array）：多个值的集合

## 转化回传

### 自动回传

配置完成后，系统可以自动将转化数据回传给广告平台：

1. 进入事件详情页
2. 开启 **自动回传** 开关
3. 选择回传目标平台
4. 配置回传规则（可选）

### 回传验证

回传后可在以下位置查看：
- **事件详情** → **回传记录**
- **广告平台** → 转化数据报表

## 事件测试

### 测试工具

在事件管理页面提供测试工具：

1. 点击 **测试事件** 按钮
2. 模拟发送测试事件
3. 查看接收结果
4. 验证参数正确性

### 调试模式

开启调试模式后，可以查看：
- 事件接收状态
- 参数解析结果
- 归因匹配过程

## 最佳实践

### 事件命名规范

- 使用小写字母和下划线
- 名称简洁明了
- 遵循团队统一的命名规范

示例：
- ✅ `app_install`
- ✅ `purchase_complete`
- ❌ `APP INSTALL`
- ❌ `Purchase-Complete`

### 参数设计原则

- 只添加必要的业务参数
- 避免收集敏感个人信息
- 保持参数命名一致性

### 事件数量建议

- 核心转化事件：3-10 个
- 辅助分析事件：10-30 个
- 避免过度细分导致分析复杂

## 常见问题

### 事件未收到数据？

1. 检查 SDK 集成是否正确
2. 查看控制台是否有错误
3. 使用测试工具验证事件配置
4. 联系技术支持

### 如何修改事件定义？

已创建的事件可以修改显示名称和描述，但事件标识符（event_id）不能修改。如需更改标识符，需要创建新事件并逐步迁移。

### 转化窗口期是什么？

转化窗口期是指从用户点击广告到发生转化行为的时间范围。常见设置为：
- 点击后转化：1-7 天
- 曝光后转化：1-24 小时

您可以在账户设置中配置窗口期。
